    Assessing the VAR of the completion time for a complex industrial project of activities using Markov chain modelling and phase-type distributions

    Industrial Problem

    Industrial projects are often marked by significant inherent complexity and variability.

    - The first aspect is represented by the numerous activities that comprise a project and the complex precedence relationships to which the activities must adhere.

    - The second point is the duration of individual activities, which is never deterministic but characterized by variability.

    For these reasons studying the KPIs of an activity project is difficult.

    Proposed Solution

    This thesis aims to provide a comprehensive methodology for studying a generic activity project in presence of variability. In particular, the completion time value at risk (VAR) will be investigated.

    The activity project is modeled as a continuous-time Markov chain, where each activity follows a phase-type distribution. The proposed methodology enables the analyticla estimation of the distribution of project duration through the solution of an ordinary differential equations system.

    Employing a newly developed Python tool, a fast assessment of the network completion time VAR is provided. This enables planners for measuring risk and assessing robust methodology to face uncertainty.

    Model Details

    This section present a detail description of the methodology used. To guide the explenation the simple network in Figure 1 is considered.

    From activity network to state space network

    The analysis starts from a general network of activities representing the project. A directed graph with precedence relations with activity on arc (AoA) representation is used as a network model. In this type of graph, the dependencies or precedence relations among activities are represented by directed edges connecting the nodes.

    The state space network corresponding to the activity network is then computed. This describes the different paths to achieve activity network completion. During the execution of the activity network, each activity can be found in one of three possible states:

    • Pending (0): This state refers to an activity that is waiting for the completion of some other preceding activity before it can start.
    • Running (1): This state indicates that the activity is currently being executed.
    • Terminated (2): This state indicates that the activity has been completed.

    A system state is a row vector containing the specific state of each activity composing the network. For the activity network in figure A1, the starting state is [1,1,0,0,0], as activities 1 and 2 start immediately while the other three activities remain pending.

    • The generation of the subsequent state is triggered only by one activity, whose current state is running. Therefore, from one set of states to the next, only one activity state will change from running to terminated. This also implies that, if the precedence state has more than one activity running in parallel, multiple sets of states will be generated.
    • Every activity in the project must be completed to achieve the project's overall goal, regardless of the path taken to complete each activity. In other words, the completion of every single activity is necessary to achieve the completion of the entire project.

    By combining the two concepts above, it is possible to create a final guideline for the state space network computation. Every activity must be completed to finish the project, but multiple activity completion paths exist. The result of this computation algorithm will always be the completion of all activities, with each path consisting of a number of steps equal to the number of activities in the project, each involving the completion of activities in a different order. Figure A2 shows the state space network for the activity network in figure A1.
    System states are represented by network nodes, while the arcs represent the activity that terminates.

    Continuous Time Markov Chain modelling

    Time is incorporated into the analysis using a continuous time Markov chain model.
    A Markov process, also known as a Markov chain, is a mathematical model used to describe the behaviour of a stochastic system over time. When describing a Markov process, a collection of interlinked stochastic processes is considered, that together model the behaviour of the entire system over time. The system moves from one state to another according to a set of transitions that can be described by continuous-time probabilities distributions that regulate the evolution of the system.
    Continuous-time Markov chain processes are governed by two fundamental properties that determine their dynamics and future evolution:

    • All activity durations are mutually independent positive random variables.
    • All activity durations distributions are characterized by the memoryless property.

    For a general activity network, under the assumption that the activity duration satisfies the properties above, the execution of the activity network can be modelled through a continuous time Markov chain. However, the Markov chain model is applicable only if the model assumptions are met. This is usually achieved through exponential distributions that fit the model requirements but are limited in the representation of general activity duration distributions for real industrial processes.
    To cope with generally distributed activity duration, phase-type distributions are introduced.
    A phase-type distribution is a probability distribution constructed by a convolution or mixture of exponential distributions. It results from a system of one or more interrelated Poisson processes occurring in sequence, or phases. The sequence in which each of the phases occurs may itself be a stochastic process. The distributions of the phase-type model represent a random variable describing the time until absorption of a Markov process with one absorbing state. Each of the states of the Markov process represents one of the phases. This is the key intuition that is used to assess activity network makespan value at risk.
    This allows for the handling and modelling of generic probability distribution functions, that present asymmetric shapes and multimodality. Figure 4 shows an example of empirical data fitting, using phase-type distributions.

    Kronecker algebra for CTMC model formalization

    The introduction of Kronecker algebra formalizes a computational strategy to build the Markov activity network (MAN) that model the execution of the initial activity network, starting from the state space network and phase-type distributions describing project activity duration.
    The dynamics of the process modelling the execution of the Markov activity network is governed by an ordinary differential equation system. The probability of a single state evolves according to the following differential equation, where matrices D and O are computed respectively using Kronecker sum and product:

    \( \frac{{d\pi(s,t)}}{{dt}} \) = \( \pi(s,t)D(s) \) + \( \sum_{\forall i:f(s,s',i)=1} \) \( \pi(s',t)\)\(O(s',s,i) \)
    • D(s) represents the aging of the phase-type distribution in state s.
    • O (s', s, i) captures instead the evolution of the network from one state to another. The dependence of O on the current state s and the subsequent state s' highlights that this term is responsible for characterizing the probability of transitioning from one state to another, with i representing the index of the activity that causes the transition.

    The dynamics captured by the O (s', s, i) term can be summarized into three main areas:

    • Completion of activity 'i': This means that activity i transitions from a running state to a terminated state.
    • Tracking of activities running during the transition: It is necessary to keep track of the activities that were running during this transition and continue to run after. This emphasizes the memory property of the model for activities that are not terminated at state s'.
    • Starting new activities triggered by the conclusion of activity i: This means that the phase-types linked to the newly started activities need to be initialized.

    The result of this computation is the infinitesimal generator matrix describing the execution of the initial activity network. In figure 4 the Markov activity network, derived from the infinitesimal generator matrix, describing the execution of the initial activity network considering activity distributed according to general phase-type distribution is reported.

    This model captures the activity precedence network behaviour considering activity duration according to phase-type distributions. This is formalized into a single infinitesimal generator matrix that describes the Markov activity network. This matrix is the base for the activity network analysis, from the point of view of different performance parameters. In this thesis, the makespan of the network has been analysed. A numerical approach that exploits the bisection methods has been implemented to evaluate the makespan value at risk of the network.

    Model implementation

    The theoretical part discussed up to this point has been implemented as four Python functions that together model the entire problem:

    • F1: Space state network computation.
    • F2: Phase-Type distribution generator.
    • F3: Infinitesimal generator matrix computation.
    • F4: Quantile computation using bisection method.

    Model validation

    To validate the phase-type approach for the generation of the infinitesimal generator matrix, Montecarlo simulation has been used to simulate the execution of different precedence relation networks. The moments of the histogram generated from the simulation have been then compared with the phase-type, moments and probability distribution computed analytically.     Model performance

    Finally, the developed approach has been tested using 92 different networks consisting of 32 activities each from PSPLIB . This has been done to assess the performance of the different functions developed in the model while dealing with different types of problems. Function 1 for computing the state space network, Function 3 for computing the infinitesimal generator matrix and Function 4 to implement the bisection method have been evaluated in terms of computational time.

    From the analysis carried out, the computation of the infinitesimal generator matrix (F3) resulted to be the slowest part of the process. Here the algorithm takes in the worst case 33 [min] to compute the infinitesimal generator matrix starting from a state space with 30.000 as dimensions. F3 shows quadratic behaviour. The state computation function (F1) takes 40 [s] in the worst case which is quite less if compared to F3. Also, the state computation function is characterized by a quadratic behaviour. Analysis of Function 4 for the solution of the exponential matrix with expv, shows promising results as the behaviour seems linear and the worst-case matrix size takes just 40 seconds to obtain a result.
